# 🎨 GUI 版本快速開始

## 🚀 3 步驟開始使用

### 1️⃣ 安裝依賴
```bash
pip install -r requirements.txt
```

> **注意**：tkinter 在 Windows 和 macOS 通常已內建。  
> Linux 用戶可能需要：`sudo apt-get install python3-tk`

### 2️⃣ 啟動 GUI
```bash
python execution\sticker_splitter_gui.py
```

### 3️⃣ 使用介面
1. 點擊「**選擇圖片**」選擇您的貼圖集
2. 確認規格設定（預設：2560×1664，4×5）
3. 點擊「**🚀 開始切割**」
4. 等待完成後，點擊「**📁 開啟輸出目錄**」查看結果

## ❓ 常見問題

### Q1: 圖片尺寸不是 2560×1664 怎麼辦？
**A:** 可以取消勾選「使用標準貼圖集規格」，然後：
- 輸入實際圖片尺寸
- 設定對應的行列數
- 程式會按比例切割

### Q2: 為什麼預覽看起來模糊？
**A:** 預覽是為了適應視窗大小而縮放的，實際切割使用原始高解析度圖片，不會影響品質。

### Q3: 可以切割其他數量的貼圖嗎？
**A:** 可以！調整行列數即可：
- 8 張：2×4 或 4×2
- 16 張：4×4
- 24 張：4×6 或 6×4
- 32 張：4×8 或 8×4
- 40 張：5×8 或 8×5

### Q4: 輸出的貼圖為什麼是 370×320 而不是 512×416？
**A:** 512×416 是貼圖集中每格的原始尺寸。程式會自動調整為 LINE 官方規格 370×320，符合上架要求。

### Q5: 支援 JPG 格式嗎？
**A:** GUI 支援 JPG 輸入，但輸出統一為 PNG（LINE 需要透明背景）。

### Q6: 程式卡住了怎麼辦？
**A:** 程式使用背景執行緒，不應該卡住。如果真的無響應：
- 檢查圖片是否過大（>8000×8000）
- 檢查可用記憶體
- 嘗試重新啟動程式

### Q7: 如何準備標準尺寸的貼圖集？
**A:** 在影像編輯軟體（如 Photoshop、GIMP）中：
1. 建立 2560×1664 px 的畫布
2. 使用參考線分割成 4×5 網格（512×416 每格）
3. 在每格中放置貼圖
4. 匯出為 PNG

## 🔧 進階技巧

### 技巧 1：批次處理
如果有多個貼圖集要處理：
1. 依序選擇每個圖片

輸出檔案自動命名為 `sticker_01.png` ~ `sticker_20.png`：
- 編號對應網格位置（從左到右，從上到下）
- 第 1 張 = 左上角第 1 格
- 第 20 張 = 右下角最後 1 格

## 🆘 疑難排解

### 問題：無法啟動 GUI
**原因**：tkinter 未安裝  
**解決**：
- Windows/Mac：重新安裝 Python（確保勾選 tkinter）
- Linux：`sudo apt-get install python3-tk`

### 問題：找不到模組 PIL
**原因**：Pillow 未安裝  
**解決**：`pip install Pillow`

### 問題：切割後檔案過大
**原因**：圖片複雜度高或透明度層次多  
**解決**：
- 簡化貼圖設計
- 減少顏色數量
- 降低透明度層次
