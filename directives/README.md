# Directives 目錄

這個目錄包含所有任務的標準操作程序（SOP）。

## 📝 指令格式

每個指令文件應該包含以下部分：

```markdown
# [任務名稱]

## 目標
明確說明這個任務要達成什麼目標

## 輸入
- 列出需要的輸入參數
- 說明數據來源

## 使用工具/腳本
- 列出要使用的執行腳本
- 說明調用順序

## 輸出
- 說明預期輸出格式
- 說明結果存放位置

## 邊界情況
- 列出可能的錯誤情況
- 說明如何處理異常

## 已知限制
- API 限制
- 時間限制
- 其他注意事項
```

## 📋 範例

參考 `example_directive.md` 了解如何編寫指令文件。

## 🔄 更新原則

當發現以下情況時，請更新相應的指令：
- API 限制或變更
- 發現更好的執行方法
- 遇到新的邊界情況
- 時間或成本優化
