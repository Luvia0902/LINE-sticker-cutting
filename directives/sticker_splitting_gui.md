# LINE 貼圖切割任務 - GUI 版本

## 目標
使用圖形化介面將一張包含多個貼圖的大圖（貼圖集）切割成獨立的貼圖圖片，並符合 LINE 貼圖規格。

## 精確規格定義

### 貼圖集標準規格
- **整體畫幅**：2560 × 1664 px（強制橫向矩形畫幅）
- **結構佈局**：4 橫排 (Rows) × 5 直欄 (Columns)
- **總貼圖數**：20 張
- **每格尺寸**：512 × 416 px（自動計算：2560÷5 × 1664÷4）

### 網格與佈局絕對定義
```
2560px (寬度)
┌─────┬─────┬─────┬─────┬─────┐
│     │     │     │     │     │ ← 416px
│  1  │  2  │  3  │  4  │  5  │
├─────┼─────┼─────┼─────┼─────┤
│     │     │     │     │     │ ← 416px
│  6  │  7  │  8  │  9  │ 10  │
├─────┼─────┼─────┼─────┼─────┤ 1664px (高度)
│     │     │     │     │     │ ← 416px
│ 11  │ 12  │ 13  │ 14  │ 15  │
├─────┼─────┼─────┼─────┼─────┤
│     │     │     │     │     │ ← 416px
│ 16  │ 17  │ 18  │ 19  │ 20  │
└─────┴─────┴─────┴─────┴─────┘
  ↑512px↑
```

### 輸出規格（LINE 官方）
- **輸出尺寸**：370 × 320 px（每張貼圖會自動調整）
- **檔案格式**：PNG（支援透明背景）
- **檔案大小**：< 1MB（自動優化壓縮）

## 輸入

### 必須輸入
- **貼圖集圖片**：
  - 路徑：使用者通過 GUI 選擇
  - 建議尺寸：2560 × 1664 px
  - 格式：PNG（推薦）或 JPG

### 可選輸入
- **輸出目錄**：預設為 `.tmp/output/stickers`
- **自訂配置**：可調整行列數（不使用標準規格時）

## 使用工具/腳本

### 主要工具
1. **GUI 應用**：`execution/sticker_splitter_gui.py`
   - 提供圖形化操作介面
   - 即時預覽與網格顯示
   - 進度指示
   - 一鍵執行切割

### 核心功能
- **圖片選擇**：檔案瀏覽器選擇輸入圖片
- **預覽顯示**：載入圖片並顯示網格線預覽
- **規格設定**：

  - 自訂：可調整任意尺寸和佈局
- **執行切割**：背景執行緒處理，不阻塞 UI
- **進度追蹤**：即時顯示處理進度

- **檔案命名**：`sticker_01.png` 到 `sticker_20.png`
- **檔案規格**：
  - 尺寸：370 × 320 px（LINE 標準）
  - 格式：PNG

## 操作流程

### 1. 啟動 GUI
```bash
python execution\sticker_splitter_gui.py
```

### 2. 選擇輸入圖片
- 點擊「選擇圖片」按鈕
- 在檔案瀏覽器中選擇貼圖集圖片
- 預覽區會自動顯示圖片和網格線

### 3. 檢查規格設定
- 預設使用標準規格（2560×1664，4×5）
- 如需自訂，取消勾選「使用標準貼圖集規格」
- 調整貼圖集尺寸和佈局參數

### 4. 設定輸出目錄（可選）
- 預設輸出至 `.tmp/output/stickers`
- 點擊「選擇目錄」可變更輸出位置

### 5. 執行切割
- 點擊「🚀 開始切割」按鈕
- 進度條顯示處理進度
- 完成後顯示結果對話框

### 6. 查看結果
- 點擊「📁 開啟輸出目錄」
- 查看切割好的 20 張貼圖

## 邊界情況

### 1. 圖片尺寸不符標準規格
- **檢查**：GUI 會顯示實際尺寸
- **處理**：系統會按比例切割，建議調整原圖尺寸
- **建議**：使用影像編輯軟體調整為 2560×1664

### 2. 輸入圖片不是 PNG
- **檢查**：GUI 支援 JPG/JPEG 格式
- **處理**：可正常切割，但輸出仍為 PNG
- **注意**：JPG 不支援透明背景

### 3. 輸出目錄不存在
- **檢查**：程式會自動檢查
- **處理**：自動建立所需目錄結構
- **確認**：完成後可點擊「開啟輸出目錄」驗證

### 4. 檔案大小超過 1MB
- **檢查**：程式會自動優化壓縮
- **處理**：如仍超過，需降低原圖複雜度
- **建議**：簡化圖案、減少透明度層次

### 5. 程式無響應
- **原因**：處理大圖需要時間
- **處理**：使用背景執行緒，不會凍結 UI
- **狀態**：查看進度條和狀態欄

### 6. 記憶體不足
- **原因**：圖片過大
- **處理**：分段處理或降低圖片尺寸
- **限制**：建議原圖不超過 5000×5000 px

## 已知限制

### 技術限制
- **最大圖片尺寸**：建議不超過 8000×8000 px（記憶體限制）
- **最大切割數**：建議不超過 100 張（處理時間考量）
- **支援格式**：PNG、JPG/JPEG（輸出統一為 PNG）

### LINE 規格限制
- **貼圖尺寸**：固定 370×320 px
- **檔案大小**：每張 < 1MB
- **套組數量**：8/16/24/32/40 張（本工具支援 20 張）
- **檔案格式**：僅 PNG（需透明背景）

### 系統需求
- **Python**：3.7+
- **依賴**：tkinter（通常內建）、Pillow
- **作業系統**：Windows、macOS、Linux
- **記憶體**：建議 2GB+ 可用記憶體

## 效能指標

### 處理時間（參考值）
- **標準規格**（2560×1664，20張）：約 5-15 秒
- **大圖**（5000×5000，40張）：約 20-60 秒
- **影響因素**：原圖大小、CPU 效能、儲存裝置速度

### 資源使用
- **CPU**：中等使用（切割和縮放時）
- **記憶體**：取決於原圖大小（約 100-500MB）
- **儲存**：視貼圖數量（每張約 50-500KB）
